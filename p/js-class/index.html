Class 란

클래스는 객체 지향 프로그래밍에서 특정 객체를 생성하기 위해 변수와 메소드를 정의하는 일종의 틀로, 객체를 정의하기 위한 상태(멤버 변수) 와 메서드(함수)로 구성된다.

객체 지향 프로그래밍에서 특정 객체를생성하기 위해 변수와 메서드를 정의하는 일종의 틀이다. new 연산자를 사용해 Class 틀을 사용하여 새로운 인스턴스를 생성하는것이 가능하며, Class(틀)을 사용해 각 인스턴스(틀로 찍어낸 도구)로 다양한 기능을 구현할 수 있다.

class Person {
  constructor(_age, _name, _mbti) {
    // Person을 구성할 틀을 만들기 위한 요소를 정의
    // 같은 class 스코프 내의 메서드에서
    // this를 통해 공통사용이 가능해지는 변수를 지정함.
    this.age = _age;
    this.name = _name;
    this._mbti = _mbti;
  }

  im() {
    // constructor 에서 구성한 요소로 Person을 소개해보자
    console.log('im ' + this.age + ' years old\nname is ' + this.name + '\nmbti is ...' + this._mbti);
  }
}

Class에서의 인스턴스란

이 포스팅 초기에서 말 한 것처럼, Class(틀)을 사용해 new 연산자로 각 인스턴스(틀로 찍어낸 도구)로 다양한 기능을 구현가능. 해당 내용을 기본 토대로 두고있다. 이해하기 쉽도록 틀 과 틀로 찍어낸 도구로 정의하였지만, 도구에 국한되지않고 다양한 사용 예시를 생각하여 보면 인스턴스에 대한 기본적인 인식이 더 쉽지않을까 싶다.

Class에서의 getter 와 setter

constructor에서 선언된 this값을 get 을 통해 return 시키고, set 을 통해 재정의, 또는 계산된 값을 인스턴스 내에서 사용하기 위해 필요한 것이 getter와 setter이다.

class Person {
  constructor(_age, _name, _mbti) {
    this.age = _age;
    this.name = _name;
    this._mbti = _mbti;
  }
  get mbti() {
    // 인스턴스 외부에서 constructor의 this 요소에 접근 / 수정할 때
    // 해당 값을 return 시키기 위해 get을 타게 됨.
    // 접근 하는 방법 => person.mbti;
    return this._mbti;
  }

  // 접근한 constructor 의 this 요소를 수정하기 위해 타는 set 영역
  set mbti(_mbti) {
    // 1 person.mbti = "INTP"로 재정의하면
    // 2 자동으로 set mbti() 를 타며
    if (_mbti.length > 5) {
      _mbti = '';
    }
    this._mbti = _mbti;
    // 3 재정의된 mbti값을 사용한 im()을 다시 호출하게됨.
    this.im();
  }
  im() {
    console.log('im ' + this.age + ' years old\nname is ' + this.name + '\nmbti is ...' + this._mbti);
  }
}
let person = new Person(60, 'john', "don't know");
person.im();

생성된 person 인스턴스에 직접 접근할 경우, Class 내에서 get 과 set 을 정의해둔 요소가 반응하게 된다.

Class 원형의 복제(extends)

클래스 상속이란, 기존에 만들어둔 Class의 constructor와 메서드를 그대로 사용하거나, 부분적으로 수정이 필요한 별개의 Class를 새로 생성하고 싶을 때 유용한 기능이다. 기존 Class를 불러와 extends 시키는 방법으로 새로이 만들수 있으나, 부분적으로 메서드와 요소를 추가하거나 수정이 필요한 경우 super()를 통해 필수로 처리해주어야 하는 부분들이 존재한다.

일단 새로 만들고싶은 extends Class를 선언 후, 인스턴스를 생성함과 동시에 기존 person 인스턴스를 생성할 때와 동일하게 인자값을 쥐어준다.

class Pianist extends Person {
  // Person Class의 모든 값과 메서드를 '상속' 받음

// name is Jan Lisiecki

직접 정의해주지않아도 Person Class의 형태를 그대로 물려받은 것을 확인할 수 있다.

하지만 앞서 말했듯, 부분적으로 메서드 또는 요소의 수정이 필요한 경우, 오버라이딩 이 필요하다. 먼저 constructor에서의 요소 추가가 필요 할 경우 생성자 오버라이딩을 알아보자.

Person Class 에서 사용하던 인자값을 받아오지 않은 채 constructor에 새로운 인자값을 넣고 기존의 인자값을 바꾸자 에러가 일어난다. 아래와 같이 기존 인자값을 super를 통해 처리해 준 후, 다시 인스턴스를 생성해보자.

class Pianist extends Person {
  // 생성자 오버라이딩 - 자체 생성자에서 "추가로 생성자를 선언".
  constructor(...args) {
    // 상속 클래스안에서 자체 생성자가 없는 경우 constructor 는 상속받은 그대로 자동으로 만들어짐
    // 상속 클래스의 생성자에는 반드시 super(...) 를 호출해야함.
    // ㄴ 추가될 this. 를 사용하기전에 반드시 사용해야함.

    // 부모 클래스에서 지정한 "매개변수들로 받아온" 상속클래스의 인자값을 호출함
    super(...args);
    this.work = 'play the piano';
    this.age = '29';
  }
}
let pianist = new Pianist(28, 'Jan Lisiecki', "don't know");
pianist.im();

Person Class 에서 사용하던 인자값을 전부 ...args 를 통해 받아온 후, super()로 정의해주었다. 에러를 내지않고 새로운 인자값(this.work)과 기존의 인자값(this.age) 모두를 사용할수 있게 되었다.

extends한 Class 안에서, 부모가 되는 기존의 Class의 메서드를 확장 사용하고 싶을 때에도 동일하다. 하지만 메서드 오버라이딩의 경우 특이한 사항이 있다. super 처리를 하지 않은 채, 동일한 이름의 메서드를 extends Class 내에서 재정의 할 경우, 부모 Class에서 상속받은 메서드가 아닌 자체 메서드를 사용하게된다.

class Pianist extends Person {
  constructor(...args) {
    super(...args);
    this.work = 'play the piano';
    this.age = '28';
  }
  im() {
    // 기존 im()의 콘솔출력을 alert으로 바꾸었다.
    alert('dingding');
  }
}
let pianist = new Pianist(28, 'Jan Lisiecki', "don't know");
pianist.im();

부모격인 Person Class에서 정의된 console.log()를 무시하고 alert만 출력하는것을 확인할 수 있다. 현재 im() 메서드를 재정의 한 것에 대해 당장은 문제가 없을지 모른다. 하지만 부모 im() 메서드의 일부 기능만 변경하거나, 확장하고 싶을 때 이에 따른 이슈가 발생하게 된다. 이때에도 constructor와 동일하게, super()를 사용하여 해결할 수 있다.

class Pianist extends Person {
  constructor(...args) {
    super(...args);
    this.work = 'play the piano';
    this.age = '28';
  }

  // 메서드 오버라이딩 - 상속 클래스내의 메서드를 "확장"하여 재사용하는 방법.
  // 부모 메서드를 토대로 일부 기능만 변경, 확장하고 싶을 때
  // 커스텀 메서드를 만들어 작업하게 되는데,
  // 이 과정 전·후에 부모 메서드를 호출하고 싶을 때가 있습니다.
  // super를 사용해서 부모클래스의 동일 메서드를 호출
  im() {
    super.im(); // 부모클래스의 im을 호출
    alert('dingding');
  }
}
let pianist = new Pianist(28, 'Jan Lisiecki', "don't know");
pianist.im();

위 코드블럭에서와 같이, 메서드 내에서 super.im()을 통해 부모 메서드를 호출할 수 있다. 해당 코드를 실행해보면, pianist.im() 에서 부모 Person.im()의 기능을 가져와 출력해준다. 결과적으로 Person Class의 im() 콘솔 + pianist.im() 고유의 alert()가 추가되어 실행되는 모습을 확인할 수 있다.

Class 원형의 복제(extends)를 위한 Prototype

이 extends를 동작하기 위한것이 바로 prototype 인데, js에서의 prototype이란 객체의 특성을 다른 객체로 상속하는것을 가능하게 하는 매커니즘이라고 한다. 기본적으로 JS의 모든 객체는 [[Prototype]] 내부 프로퍼티를 가진다. 자신에게 연결된 프로토 타입 객체에 대한 참조이다. 이 prototype 을 확인하기 위해서는 아무 변수나 선언 한 후, 해당 변수에 .__proto__ 를 붙여 확인해보면 된다. 각 변수의 형태별로 전부 고유의 프로토타입을 가지고 있는 모습이 보인다. Class extends 를 진행하기위해 해당 prototype들을 관찰해보자.

let string = new String();
console.log(string.__proto__);
let obj = new Object();
console.log(obj.__proto__);
let array = [];
console.log(array.__proto__);

단지 String / Object / Array 를 새로 정의하였을 뿐인데, 각자 고유의 prototype을 가지고있는 모습을 확인할 수 있다.
해당 프로토타입 원리가, Class의 extends 생성을 위해서도 사용되는 모습을 확인할 수 있다.

  1. Pianist.prototype.[[Prototype]] 을 Person.prototype 으로 설정함.
  2. Pianist.prototype 에서 메서드를 찾지 못하면
  3. Person.prototype 에서 메서드를 가져옴.

[예시]

  1. 기존에 Class Person이 정의되어있는 상태일 경우를 가정
  2. Person 클래스를 가지고, extends 한 let pianist = new Pianist(); 생성시
  3. Pianist.prototype 에 im() 메서드가 있는지 확인함.
    3-1. if - im() 메서드가 있을 경우, Class 에 선언되어있는 해당 메서드를 그대로 사용한다.
    3-2. else if - im() 메서드가 없을 경우, extends 를 통해 관계가 만들어진 Person.prototype에 im() 메서드가 있는지 확인한다.
    3-2-1. im() 메서드를 찾은 후, 객체 pianist 에 im() 메서드를 추가해준다.
    ㄴ 이 과정에서, 메서드 오버라이딩 super() 처리가 되어있을 경우에만 부모 클래스의 im() 메서드를 불러 올 수 있게 된다.

Class 문법 사용 예시

해당 내용들을 토대로 amCharts5 를 이용한 차트 하나를 구현해보았다. 인스턴스에서 받은 파라미터를 통해 데이터를 받아오는데, 데이터가 바뀔 때에만 set을 통하여 차트를 새로 그려주는 방식을 채택했다. 간략한 순서는 이러하다.

  1. <form>내의 <select> 태그가 가지고 있는 value값을 받음.
  2. value값을 통해 나의 깃헙 레포지토리에서 조건에 맞는 JSON을 fetch를 타고 불러옴.
  3. fetch 내에서
    3-1. JSON 파일 array의 object 인덱스를 선택할 버튼을 그려줌.
    => 버튼을 그려줄 Class 사용함
    3-2. JSON 파일 array 첫번째 인덱스의 데이터를 그대로 차트에 뿌려주게끔 설정.
    => 차트를 그려줄 Class 사용함
  4. 버튼 클릭시 차트를 그려주는 data 인자값을 인스턴스에 새로 넘김.
  5. 차트드로잉 인스턴스의 set data를 타고, 차트를 새로 그려줌.
    링크: https://songjung110.github.io/uxLab-task-song/js-class-chart-playground/

아직 구현하지 못했지만 extends를 잘 사용하면, 동일한 데이터로 형태가 다른 여러 차트를 그려주는 확장 Class를 만들어볼수 있지 않을까 싶다.
